05 — Programmer

Programmer

La chaîne logicielle se décompose en deux grandes parties : la génération du G-code côté PC (avec Processing), et l'interprétation côté microcontrôleur. L'utilisation de Processing pour générer le G-code est une exigence du projet.

Flux de données — de l'idée au tracé

[1. Création du contenu — PC]
  Fichier numérique : dessin vectoriel (SVG, DXF) ou bitmap (PNG, JPG)
  → Logiciel de traitement : Processing ou Inkscape
  → Vectorisation si besoin (bitmap → SVG)
  → Génération du G-code : liste de coordonnées (X, Y) et instructions de mouvement

[2. Transmission — PC → ESP32]
  Envoi via port série (USB)
  Le logiciel "Sender" envoie les lignes de G-code une par une

[3. Traitement — ESP32 + Firmware]
  Réception et interprétation du G-code
  Calcul du nombre de pas STEP/DIR par axe pour atteindre chaque coordonnée cible
  Génération des signaux électriques

[4. Amplification — Drivers A4988]
  Le signal logique ESP32 (trop faible) est amplifié
  Le driver puise dans l'alimentation 12V externe
  → Impulsions de courant haute intensité vers les bobines des moteurs

[5. Action mécanique — NEMA 17 + Courroies]
  Rotations précises des moteurs ("pas")
  Transmises via courroies GT2 aux chariots
  → Le stylo trace le dessin sur la feuille

Commandes G-code supportées

CommandeAction
G0 X__ Y__Déplacement rapide sans tracé (stylo levé)
G1 X__ Y__ F__Déplacement avec tracé (stylo baissé). F = vitesse en mm/min
M3Baisser le stylo
M5Lever le stylo

Firmware — code principal

// Machine That Draw — Groupe 06
// Makerspace Amiens 2025-2026

#include <Stepper.h>
#include <Servo.h>

const int STEPS = 200;
Stepper motorX(STEPS, 8, 9, 10, 11);
Stepper motorY(STEPS, 4, 5, 6, 7);
Servo penServo;

void setup() {
  Serial.begin(115200);
  motorX.setSpeed(60);
  motorY.setSpeed(60);
  penServo.attach(12);
  penServo.write(90); // Stylo levé
}

void loop() {
  if (Serial.available()) {
    String cmd = Serial.readStringUntil('\n');
    parseGCode(cmd);
  }
}

Conversion G-code avec Processing

1
Préparation du fichier source
Dessin vectoriel (SVG, DXF) ou image bitmap (PNG, JPG). Les SVG sont directement utilisables ; les bitmaps nécessitent une étape de vectorisation.
2
Vectorisation (si nécessaire)
Si le fichier est une image bitmap, on le convertit d'abord en SVG via Inkscape ou un outil équivalent.
3
Génération des trajectoires avec Processing
Le programme Processing lit le fichier vectoriel, calcule l'ordre de parcours optimisé des contours et génère les coordonnées X/Y.
4
Export G-code
Le fichier .gcode est généré avec les commandes G0/G1/M3/M5. Il est prêt à être envoyé en série à l'ESP32.
5
Envoi vers l'ESP32
Le logiciel Sender envoie les lignes de G-code une par une via USB (115200 bauds). L'ESP32 exécute chaque instruction au fur et à mesure.

Exemple de G-code généré

; Dessin d'un carré 50×50 mm
G0 X0 Y0         ; Aller à l'origine
M3               ; Baisser le stylo
G1 X50 Y0 F800   ; Ligne horizontale 50 mm
G1 X50 Y50       ; Ligne verticale 50 mm
G1 X0 Y50        ; Ligne horizontale retour
G1 X0 Y0         ; Fermer le carré
M5               ; Lever le stylo
G0 X0 Y0         ; Retour origine
