Conversion G-code

Avant d'envoyer un dessin à la machine, le fichier source doit être converti en une liste d'instructions G-code que le firmware peut interpréter. La chaîne retenue utilise Processing pour la génération, conformément aux exigences du projet.

Processus de conversion

1
Préparation du fichier source
Dessin vectoriel (SVG, DXF) ou image bitmap (PNG, JPG). Les SVG sont directement utilisables ; les bitmaps nécessitent une étape de vectorisation.
2
Vectorisation (si nécessaire)
Si le fichier est une image bitmap, on le convertit d'abord en SVG via Inkscape ou un outil équivalent.
3
Génération des trajectoires avec Processing
Le programme Processing lit le fichier vectoriel, calcule l'ordre de parcours optimisé des contours et génère les coordonnées X/Y.
4
Export G-code
Le fichier .gcode est généré avec les commandes G0/G1/M3/M5. Il est prêt à être envoyé en série à l'ESP32.
5
Envoi vers l'ESP32
Le logiciel Sender envoie les lignes de G-code une par une via USB (115200 bauds). L'ESP32 exécute chaque instruction au fur et à mesure.

Outils utilisés

Exemple de G-code généré

; Dessin d'un carré 50×50 mm
G0 X0 Y0         ; Aller à l'origine
M3               ; Baisser le stylo
G1 X50 Y0 F800   ; Ligne horizontale 50 mm
G1 X50 Y50       ; Ligne verticale 50 mm
G1 X0 Y50        ; Ligne horizontale retour
G1 X0 Y0         ; Fermer le carré
M5               ; Lever le stylo
G0 X0 Y0         ; Retour origine

Paramètres importants

Unitésmm
Vitesse d'avance (F)✏️ ex : 800 mm/min
Commande stylo baisséM3
Commande stylo levéM5
Déplacement rapide (sans tracé)G0
Déplacement tracéG1
